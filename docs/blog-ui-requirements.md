# 个人技术博客前端需求文档

## 1. 项目概述

本项目用于构建一个基于 Vue 3、Vite 与 Element Plus 的个人技术博客前端界面。页面以个人技术内容展示为核心，兼顾账户信息展示、标签筛选、文章阅读入口、问答互动或扩展信息展示。

目标是提供一个结构清晰、阅读舒适、可扩展的博客首页，为后续接入 Spring Boot 后端接口、文章详情页、评论系统、搜索系统等功能预留空间。

## 2. 技术选型

- 前端框架：Vue 3
- 构建工具：Vite
- UI 组件库：Element Plus
- 图标库：@element-plus/icons-vue
- HTTP 请求：Axios
- 样式方案：CSS / Scoped CSS，后续可按需要扩展为 CSS Variables 或 Sass

## 3. 页面整体结构

页面采用顶部账户信息区 + 主体三栏布局。

### 3.1 顶部个人信息区

顶部区域用于展示博客作者的身份信息，建议保持简洁、醒目。

核心内容：

- 个人头像
- 昵称 / 用户名
- 职业或技术方向说明
- 简短个人签名
- 文章数、标签数、问答数等统计信息
- 可选操作入口：GitHub、邮箱、关于我、后台管理入口

建议组件：

- `el-avatar`
- `el-tag`
- `el-button`
- `el-space`
- `el-statistic`

### 3.2 主体三栏布局

主体内容分为左、中、右三栏：

- 左侧：标签栏
- 中间：文章内容栏
- 右侧：问答、动态、推荐或其他扩展区域

桌面端推荐布局比例：

- 左侧栏：220px - 260px
- 中间内容栏：自适应，作为主阅读区域
- 右侧栏：280px - 340px

移动端布局：

- 顶部信息区保持优先展示
- 左侧标签栏可折叠为横向滚动标签或抽屉
- 中间文章列表优先展示
- 右侧栏移动到文章列表下方

## 4. 功能需求

### 4.1 个人信息展示

页面加载时请求个人资料接口并展示作者信息。

展示字段：

- 头像地址
- 昵称
- 账户名
- 简介
- 技术栈标签
- 文章总数
- 标签总数
- 问答总数
- 外部链接

交互要求：

- 头像加载失败时显示默认头像
- 外部链接点击后新窗口打开
- 个人信息接口异常时展示默认占位信息

### 4.2 标签栏

左侧标签栏用于展示文章分类或技术标签，支持按标签筛选文章。

功能点：

- 展示全部标签
- 支持选中标签
- 支持“全部文章”选项
- 标签显示文章数量
- 当前选中标签高亮
- 标签过多时支持滚动

交互要求：

- 点击标签后刷新中间文章列表
- 重复点击当前标签可保持当前筛选状态
- 接口加载中时展示骨架屏或加载状态
- 标签接口失败时展示错误提示与重试入口

### 4.3 内容栏

中间内容栏是页面核心区域，用于展示文章列表。

文章卡片字段：

- 文章标题
- 文章摘要
- 封面图，可选
- 标签列表
- 发布时间
- 更新时间，可选
- 阅读量
- 点赞数，可选
- 评论数，可选

功能点：

- 分页加载文章列表
- 按标签筛选文章
- 支持文章卡片点击进入详情页，详情页可后续实现
- 支持空状态展示
- 支持加载状态展示
- 支持接口异常提示

建议组件：

- `el-card`
- `el-skeleton`
- `el-empty`
- `el-pagination`
- `el-tag`

### 4.4 右侧扩展区域

右侧区域用于承载问答、推荐阅读、近期动态或其他轻量内容。当前优先规划为问答区域。

问答卡片字段：

- 问题标题
- 简短描述
- 标签
- 回答数
- 更新时间

功能点：

- 展示最近问答列表
- 点击问答项进入详情页，详情页可后续实现
- 支持空状态
- 支持加载状态
- 预留切换为“推荐文章 / 最近动态 / 站点公告”的扩展能力

建议组件：

- `el-tabs`
- `el-card`
- `el-list` 可用自定义列表替代
- `el-badge`
- `el-empty`

## 5. 数据接口需求

前端默认通过 `/api` 访问后端接口，本地开发时由 Vite 代理到后端服务。

### 5.1 获取个人信息

接口：

```http
GET /api/profile
```

返回示例：

```json
{
  "nickname": "CodeChronicles",
  "account": "code_chronicles",
  "avatar": "https://example.com/avatar.png",
  "bio": "记录前端、后端与工程化实践。",
  "skills": ["Vue", "Element Plus", "Spring Boot"],
  "articleCount": 42,
  "tagCount": 12,
  "questionCount": 8,
  "links": [
    {
      "label": "GitHub",
      "url": "https://github.com/example"
    }
  ]
}
```

### 5.2 获取标签列表

接口：

```http
GET /api/tags
```

返回示例：

```json
[
  {
    "id": 1,
    "name": "Vue",
    "articleCount": 12
  }
]
```

### 5.3 获取文章列表

接口：

```http
GET /api/articles?page=1&pageSize=10&tagId=1
```

参数说明：

- `page`：当前页码
- `pageSize`：每页数量
- `tagId`：标签 ID，可选

返回示例：

```json
{
  "total": 42,
  "list": [
    {
      "id": 1,
      "title": "Vue 3 组合式 API 实践",
      "summary": "介绍组合式 API 在业务组件中的使用方式。",
      "cover": "https://example.com/cover.png",
      "tags": ["Vue", "Frontend"],
      "publishedAt": "2026-06-01",
      "updatedAt": "2026-06-01",
      "views": 1024,
      "likes": 88,
      "comments": 12
    }
  ]
}
```

### 5.4 获取问答列表

接口：

```http
GET /api/questions
```

返回示例：

```json
[
  {
    "id": 1,
    "title": "Element Plus 表格如何做服务端分页？",
    "description": "记录一次表格分页、筛选与排序联动的实现。",
    "tags": ["Element Plus", "Vue"],
    "answerCount": 3,
    "updatedAt": "2026-06-01"
  }
]
```

### 5.5 通用响应结构

接口可以直接返回数据，也可以使用统一响应结构：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

前端请求层需要兼容以上两种返回方式。

## 6. 状态与异常处理

页面需要覆盖以下状态：

- 首次加载状态
- 局部模块加载状态
- 空数据状态
- 接口错误状态
- 网络异常状态
- 图片加载失败状态

推荐处理方式：

- 文章列表加载中使用 `el-skeleton`
- 无文章时使用 `el-empty`
- 请求失败时使用 `ElMessage.error`
- 局部区域失败时保留页面结构并提供重试按钮

## 7. 响应式需求

### 7.1 桌面端

- 三栏完整展示
- 中间文章区域为视觉中心
- 左右栏随页面滚动保持自然布局，可后续扩展为吸顶

### 7.2 平板端

- 保留中间内容栏和右侧栏
- 左侧标签栏可缩窄或改为横向标签组

### 7.3 移动端

- 单列布局
- 标签栏变为横向滚动或抽屉
- 右侧问答区域移动到文章列表下方
- 顶部个人信息区压缩展示，避免占用过多首屏高度

## 8. 视觉与交互风格

整体风格应体现技术博客的清爽、可靠和阅读友好。

设计要求：

- 页面背景浅色、干净
- 内容区域层级清晰
- 卡片间距统一
- 标签颜色克制，避免过度花哨
- 文章标题突出但不过分夸张
- 鼠标悬停文章卡片时提供轻微反馈
- 页面首屏能同时看到个人信息与部分文章内容

## 9. 非功能需求

- 首屏加载应尽量轻量
- 请求方法统一封装，便于后续替换接口
- 组件结构清晰，便于拆分维护
- 文案与默认数据应集中管理或易于替换
- 样式应避免强耦合后端数据结构
- 后续可以扩展文章详情、搜索、归档、评论、后台管理入口

## 10. 建议组件拆分

建议拆分为以下组件：

- `ProfileHeader`：顶部个人信息区
- `TagSidebar`：左侧标签栏
- `ArticleList`：中间文章列表
- `ArticleCard`：文章卡片
- `QuestionPanel`：右侧问答面板
- `LoadingState`：通用加载状态，可选
- `EmptyState`：通用空状态，可选

建议 API 模块：

- `getProfile()`
- `getTags()`
- `getArticles(params)`
- `getQuestions()`

## 11. 验收标准

- 页面可正常运行在 Vue 3 + Vite 项目中
- 顶部能够展示头像、账户信息、简介和统计数据
- 左侧能够展示标签并支持标签筛选
- 中间能够展示文章列表、分页、加载、空状态和错误状态
- 右侧能够展示问答或扩展内容
- 页面在桌面端为三栏布局
- 页面在移动端不出现横向溢出，核心内容可正常阅读
- 接口异常时页面不崩溃，并给出合理提示
- 代码结构便于后续拆分为独立组件和接入真实后端
